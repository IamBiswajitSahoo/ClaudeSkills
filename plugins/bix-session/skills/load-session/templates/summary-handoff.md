Summarize this conversation as a structured handoff document, so someone (or a future session) can pick up exactly where this left off. Use the SOAP-inspired format below.

**Subjective — What the user wanted:**
- The user's stated goals and motivations
- Any preferences, constraints, or context they provided
- Their level of expertise or familiarity with the topic (if apparent)

**Objective — What actually happened:**
- Concrete actions taken (files changed, commands run, research done)
- Measurable results (tests passed/failed, errors encountered, data gathered)
- Tools and resources used

**Assessment — Analysis and conclusions:**
- What worked and what didn't
- Key technical findings or discoveries
- Risks, blockers, or concerns identified
- Decisions made and their rationale

**Plan — What comes next:**
- Explicit next steps mentioned by the user or agreed upon
- Open questions that need answers before proceeding
- Dependencies or prerequisites for the next phase
- Suggested approach for continuing (if discussed)

Rules:
- Write as if briefing someone who has zero context about this session
- Be specific: name files, functions, tools, URLs, error messages
- In the Plan section, prioritize actionable next steps over vague suggestions
- If the session ended mid-task, clearly state what's incomplete and what's blocking it
- Keep each section to 3-7 bullet points — comprehensive but scannable
