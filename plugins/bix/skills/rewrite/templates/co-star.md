# CO-STAR Framework

**Best for:** Content creation, writing tasks, communications, marketing copy — any task where audience, tone, and style matter.

## Template Structure

Rewrite the prompt into these sections:

### Context
Provide the background information — the situation, project, or problem. What does the AI need to know to produce relevant output?

### Objective
State the specific goal — what should be achieved? What is the deliverable?

### Style
Specify the writing style — e.g., "like a technical blog post", "academic paper", "casual tutorial", "enterprise documentation". Reference a known style if applicable.

### Tone
Set the emotional register — confident, empathetic, authoritative, friendly, urgent, neutral, humorous. This shapes how the message feels to the reader.

### Audience
Define who will read or consume the output — developers, executives, beginners, customers, internal team. This drives vocabulary, depth, and assumptions about prior knowledge.

### Response
Specify the desired format and structure — length, format (markdown, plain text, HTML), sections to include, or structural requirements.

## Rules for This Framework
- Style and Tone are distinct: Style is how it's written, Tone is how it feels
- Audience is critical — it determines the level of detail and jargon
- If the original prompt doesn't specify audience/tone, infer from context and mark with [inferred]
