# C.R.E.A.T.E. Framework

**Best for:** Content generation with fine control over style, format, and examples. Creative tasks, documentation, tutorials.

## Template Structure

Rewrite the prompt into these sections:

### Character
Define the AI persona — the role, expertise, and voice it should adopt for this task.

### Request
State the clear, specific task — what exactly needs to be produced. Use precise language.

### Examples
Provide sample inputs and outputs that illustrate the desired result. If the original prompt lacks examples, infer a brief one that demonstrates the expected quality and format, and mark it with [inferred].

### Adjustments
Specify refinements — point of view, style constraints, things to emphasize or de-emphasize, deviations from the default approach.

### Type of Output
Define the exact format — blog post, code snippet, JSON schema, markdown document, bullet list, table, etc. Include length or structural requirements.

### Extras
Add any supplementary requirements — reference materials to consider, related context, edge cases to handle, or secondary goals.

## Rules for This Framework
- Examples are the strongest lever — even a brief one dramatically improves output quality
- Adjustments capture the nuances that other frameworks miss
- Type of Output should be specific — "a document" is too vague; "a markdown guide with H2 sections" is good
