# BAB Framework

**Best for:** Transformation tasks, refactoring, migration, upgrading, rewriting — any task where you're going from a current state to a desired state.

## Template Structure

Rewrite the prompt into these sections:

### Before
Describe the current state — what exists now, what's wrong with it, or what needs to change. Be specific about the starting point: the existing code, content, system, or situation.

### After
Describe the desired state — what it should look like when done. Be specific about the end result: the target behavior, structure, quality, or outcome.

### Bridge
Explain how to get from Before to After — the approach, steps, constraints, or methodology. This is where the actual task instructions live.

## Rules for This Framework
- Before and After should create a clear contrast — the AI needs to understand the delta
- Bridge should be actionable, not vague — include specific techniques or constraints
- This framework is especially strong for refactoring prompts where "change X to Y" is the core ask
