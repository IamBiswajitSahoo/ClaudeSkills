# K.E.R.N.E.L. Framework

**Best for:** Meta-validation and refinement of any prompt. Works as a checklist to improve clarity, reproducibility, and focus rather than imposing a new structure.

## Template Structure

Apply each K.E.R.N.E.L. criterion to the original prompt and output the refined version:

### Keep it Simple
Remove unnecessary complexity, jargon, and ambiguity. Ensure the AI knows exactly what is being asked. Simplify compound requests into clear statements.

### Easy to Verify
Make success criteria explicit. Add verifiable conditions — what does a correct output look like? How can the user check if the result is right?

### Reproducible
Remove time-sensitive references, vague qualifiers ("good", "nice", "appropriate"), and implicit assumptions. Pin specifics so the prompt produces consistent results across runs.

### Narrow Scope
Tighten the focus to one clear job. If the original prompt asks for multiple things, separate them or explicitly prioritize. Remove tangential requests.

### Explicit Constraints
State all constraints directly — format, length, language, technology, things to avoid, things to include. Don't leave constraints implicit.

### Logical Structure
Organize the prompt in a coherent flow: context first, then task, then constraints, then output format. Ensure sections follow a natural reading order.

## Rules for This Framework
- This is a refinement pass, not a structural rewrite — preserve the original prompt's voice and approach
- The output should read as a single cohesive prompt, not as a checklist
- Only add what improves the prompt — do not pad with unnecessary sections
