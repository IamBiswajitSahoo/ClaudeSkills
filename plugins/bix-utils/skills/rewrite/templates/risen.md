# RISEN Framework

**Best for:** Structured multi-step tasks, detailed reports, systematic procedures, complex implementations.

## Template Structure

Rewrite the prompt into these sections:

### Role
Define who the AI should be — the expertise, perspective, and professional identity it should assume. Derive this from the domain implied by the user's prompt.

### Instructions
State clearly what needs to be done. This is the core task — be specific about the deliverable.

### Steps
Break the task into a numbered sequence of steps. Each step should be a concrete action. If the original prompt implies a workflow, make it explicit. If not, decompose the task logically.

### End Goal
Describe the desired outcome — what does success look like? What should the final output contain, accomplish, or enable?

### Narrowing
Add constraints and boundaries: what to exclude, quality standards, format requirements, scope limits, technology constraints, or any guardrails that keep the output focused.

## Rules for This Framework
- Steps should be actionable and sequenced — each one builds on the previous
- End Goal should be measurable or verifiable where possible
- Narrowing prevents scope creep — if the original prompt is broad, tighten it here
