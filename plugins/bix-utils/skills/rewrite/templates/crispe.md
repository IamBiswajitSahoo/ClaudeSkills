# CRISPE Framework

**Best for:** Tasks requiring multiple output variants, A/B testing, creative exploration, or when you want the AI to experiment with different approaches.

## Template Structure

Rewrite the prompt into these sections:

### Capacity
Define what role and expertise the AI should assume — the professional capacity it operates in.

### Insight
Provide the background knowledge, context, and information the AI needs to work with. Include relevant data, constraints, or prior decisions.

### Statement
State the core task — the specific request or instruction. This is what the AI must deliver.

### Personality
Describe the tone, style, and communication approach — how the output should feel to the reader.

### Experiment
Encourage the AI to generate multiple variants or explore alternative approaches. Specify how many variants, what dimensions should vary (tone, approach, detail level), or what trade-offs to explore.

## Rules for This Framework
- The Experiment component is what makes CRISPE unique — use it to get diverse outputs
- If the original prompt doesn't suggest experimentation, add a reasonable experiment dimension (e.g., "provide a concise and a detailed version")
- Personality and Experiment together enable rich creative exploration
